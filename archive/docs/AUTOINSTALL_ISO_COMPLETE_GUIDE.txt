# 🚀 MJR-EHR AUTOINSTALL ISO - COMPLETE GUIDE

## WHAT IS THIS?

A fully automated Ubuntu Server installation that:
- ✅ Installs Ubuntu Server 24.04 (no Desktop overhead)
- ✅ Installs Node.js, Chromium, and all dependencies
- ✅ Extracts and configures MJR-EHR automatically
- ✅ Creates systemd service for auto-start
- ✅ Configures auto-login and kiosk mode
- ✅ Boots directly to MJR-EHR fullscreen
- ✅ **ZERO user interaction required**

## CREATING THE AUTOINSTALL ISO

### Prerequisites

System running the build script needs:
- Ubuntu 22.04 or 24.04 (build host)
- Root access (sudo)
- 10GB free disk space
- Internet connection (downloads Ubuntu ISO ~2GB)

### Build the ISO

```bash
cd /home/claude/mjr-ehr-interactive/scripts
sudo bash create-autoinstall-iso.sh
```

### What It Does

1. Downloads Ubuntu Server 24.04 ISO (~2GB) if not present
2. Installs required tools (xorriso, isolinux, etc.)
3. Extracts Ubuntu ISO
4. Packages MJR-EHR application with all dependencies
5. Creates autoinstall configuration (user-data)
6. Embeds MJR-EHR into ISO
7. Configures GRUB for zero-touch installation
8. Generates new bootable ISO: `mjr-ehr-autoinstall.iso`

**Time: ~10-15 minutes** (mostly downloading Ubuntu ISO)

**Output:** `/home/claude/mjr-ehr-autoinstall.iso` (~2.5GB)

## DEPLOYING THE AUTOINSTALL ISO

### Option 1: Flash to USB Drive

#### Windows (Rufus)

1. Download Rufus: https://rufus.ie
2. Insert USB drive (8GB+)
3. Open Rufus
4. Device: Select your USB
5. Boot selection: Click SELECT → choose `mjr-ehr-autoinstall.iso`
6. Partition scheme: GPT
7. Target: UEFI
8. Click START
9. Wait for completion (~5 minutes)

#### Mac/Linux (dd)

```bash
# Find USB device
lsblk

# Flash ISO (replace /dev/sdX with your USB device)
sudo dd if=/home/claude/mjr-ehr-autoinstall.iso of=/dev/sdX bs=4M status=progress

# Sync
sync
```

#### Linux (Etcher)

1. Download: https://www.balena.io/etcher
2. Select mjr-ehr-autoinstall.iso
3. Select USB drive
4. Flash

### Option 2: Virtual Machine (Testing)

#### VirtualBox

1. Create new VM
2. Type: Linux
3. Version: Ubuntu (64-bit)
4. Memory: 4GB minimum
5. Disk: 40GB minimum
6. Settings → Storage → Add optical drive
7. Select mjr-ehr-autoinstall.iso
8. Start VM
9. Installation runs automatically

#### VMware

1. Create new VM
2. Select "I will install the operating system later"
3. Guest OS: Linux → Ubuntu 64-bit
4. Disk: 40GB minimum
5. Edit settings → CD/DVD → Use ISO image
6. Browse to mjr-ehr-autoinstall.iso
7. Power on
8. Installation runs automatically

## TROUBLESHOOTING

### Installation Fails or Hangs

**Problem:** Installation stops or shows errors

**Solutions:**
1. Check BIOS/UEFI settings:
   - Disable Secure Boot
   - Enable UEFI boot mode
   - Enable virtualization (VT-x/AMD-V)

2. Check minimum requirements:
   - 2GB RAM minimum (4GB recommended)
   - 20GB disk minimum (40GB recommended)
   - Internet connection for npm packages

3. Try in Virtual Machine first to test ISO

### System Boots to Command Line

**Problem:** No graphical interface, just login prompt

**Solutions:**

```bash
# Check if X is running
ps aux | grep X

# Check MJR-EHR service
systemctl status mjr-ehr.service

# Check logs
journalctl -u mjr-ehr.service

# Manually start X
startx
```

### Black Screen After Boot

**Problem:** System boots but screen stays black

**Solutions:**

```bash
# Switch to TTY2
Ctrl + Alt + F2

# Login as demo / demo1234

# Check if Chromium crashed
ps aux | grep chromium

# Restart X
killall X
# Logout and login again (will restart X)
```

### MJR-EHR Not Loading

**Problem:** Chromium opens but shows connection error

**Solutions:**

```bash
# Check if server is running
systemctl status mjr-ehr.service

# Check server logs
journalctl -u mjr-ehr.service -n 50

# Check if port 3000 is listening
netstat -tulpn | grep 3000

# Manually start server for debugging
cd /opt/mjr-ehr
node server/server.js
```

### Network Issues

**Problem:** No internet connectivity

**Solutions:**

```bash
# Check IP address
ip addr

# Check if DHCP worked
cat /etc/netplan/*.yaml

# Restart networking
sudo netplan apply

# Test connectivity
ping -c 4 8.8.8.8
```

## PRODUCTION HARDENING

### Security Improvements

```bash
# Change default password
sudo passwd demo

# Disable auto-login (edit getty service)
sudo rm /etc/systemd/system/getty@tty1.service.d/autologin.conf
sudo systemctl daemon-reload

# Enable firewall
sudo ufw allow 22/tcp
sudo ufw allow 3000/tcp
sudo ufw enable

# Update system
sudo apt update && sudo apt upgrade -y
```

### Enable HTTPS

```bash
# Install nginx
sudo apt install nginx certbot python3-certbot-nginx

# Configure reverse proxy
sudo nano /etc/nginx/sites-available/mjr-ehr

# Add:
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

# Enable site
sudo ln -s /etc/nginx/sites-available/mjr-ehr /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

# Get SSL certificate
sudo certbot --nginx -d your-domain.com
```

## FAQ

**Q: Can I change the hostname/username/password?**
A: Yes, edit variables in `create-autoinstall-iso.sh` before building.

**Q: Will this work on any x86_64 computer?**
A: Yes, any UEFI-capable x86_64 system. May need BIOS tweaks for older hardware.

**Q: Can I use this in production?**
A: Yes, but apply security hardening first (change passwords, firewall, HTTPS).

**Q: Does it need internet during installation?**
A: Yes, for npm packages. Most dependencies are embedded but some download.

**Q: Can I pre-configure Claude API key?**
A: Yes, add to user-data late-commands before building ISO.

**Q: How do I update MJR-EHR after installation?**
A: SSH in, `cd /opt/mjr-ehr && git pull && npm install && npm run build && sudo systemctl restart mjr-ehr`

**Q: Can I install on multiple machines simultaneously?**
A: Yes! Flash ISO to multiple USBs, boot all machines, walk away.

**Q: What if installation fails?**
A: Check BIOS settings, try in VM first, check internet connectivity.

**Q: Can I customize the autoinstall process?**
A: Yes, edit user-data section in `create-autoinstall-iso.sh`

**Q: Does it support WiFi?**
A: Wired ethernet recommended. WiFi requires additional configuration.
