3. Fixes implemented this session

S-C4 hardening (RBAC + HIPAA middleware)

Both files now use a single explicit identity-resolution helper with documented precedence:

  1. req.user — JWT identity set by auth.requireAuth
  2. req.session.userRole — validated session
  3. Header bypass — only when NODE_ENV=development AND ENABLE_DEV_AUTH_BYPASS=true
  4. Default: guest (rejected by requireRole)

The unset-NODE_ENV "default to physician" path is gone. Header forgery is impossible in production even if upstream middleware were ever misordered.

Verifier script (scripts/ehr_harden_verify_skill.py)

Read-only Python stdlib script implementing the 05-03 plan's CLI contract. Twelve check groups (A–L) plus a secret-leakage scan. Self-derives the canonical module count from MODULE_ORDER so the check stays valid as the registry evolves.

  --include-autobetter --strict

Exit codes: 0 all-pass, 1 any FAIL (or any WARN under --strict), 2 usage error.

MODULE_CATALOG.md now documents all 13 modules with per-module safety boundaries and a clear "10 encounter + 3 governance" split. README.md, VISION.md, and package.json all use the same count. The verifier will flag any future drift in --strict mode.

Auth unit tests (test/unit/auth-identity.test.js)

7. Codex coordination — do-not-collide note

Codex's 6 PM scope is HTTP testing discipline. To avoid stepping on each other this session deliberately did not touch:

  • scripts/http-hardening-smoke.js
  • test/run-tests.js (the 17-phase HTTP harness)
  • Any route/handler that an HTTP test would exercise (no signature changes)

What Codex can rely on from this session:

  • RBAC headers no longer accepted in production. If a Codex HTTP test was passing identity via x-user-role without a JWT and without setting ENABLE_DEV_AUTH_BYPASS=true, that test will now 403. Set both env flags in the test harness.
  • JWT_SECRET is not exported. Tests that signed tokens via auth.JWT_SECRET need to use auth.signToken({...}).
  • The verifier exists. Codex's HTTP discipline can call it as a pre-test gate: python scripts/ehr_harden_verify_skill.py --path . --strict — exit 1 means "fix structural drift before running HTTP tests".
